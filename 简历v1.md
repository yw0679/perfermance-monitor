项目描述

项目采用 `Worker-Manager` 架构：`Worker` 端按周期采集 CPU、负载、内存、磁盘、网络等核心指标，通过 `Protobuf` 统一封装后经 `gRPC` 上报至 `Manager`；`Manager` 端负责接收数据、维护主机状态、计算综合评分，并将监控结果持久化到 MySQL，支持按主机与时间范围进行基础查询和统计分析。

### 主要内容

1. 设计并实现 `Worker-Manager` 双端监控架构，支持通过命令行参数配置采样/上报周期,能够在单机双进程或多机部署场景下运行。
2. 通过 `/proc/stat` 结合两次采样差值计算 CPU 各态使用率，通过 `/proc/loadavg` 获取系统负载，通过 `/proc/meminfo` 统计总内存/空闲内存/可用内存及缓存信息，通过 `/proc/net/dev` 计算网卡收发速率与包速率，通过 `/proc/diskstats` 计算磁盘吞吐、IOPS、时延与利用率。
3. 使用 `Protobuf` 定义统一监控消息 `MonitorInfo`，封装主机标识、CPU、内存、网络、磁盘等数据结构；基于 `gRPC` 实现 `Worker -> Manager` 的远程上报链路。
4. 在 `Manager` 侧实现主机状态维护与评分逻辑，按 `主机名 + IP` 维度管理节点最新状态，综合 CPU、内存、负载、磁盘、网络指标计算主机评分，并将汇总数据与网络/磁盘/内存明细写入 MySQL，形成时间序列监控数据。
5. 基于 MySQL 数据表与 `QueryService` 实现基础查询能力，支持性能记录分页查询、趋势聚合、异常筛选、评分排序、最新评分汇总以及网络/磁盘/内存明细查询，初步具备“采集 + 存储 + 查询”的后端服务能力。

- 内核模块采集逻辑在仓库里有保留代码，但当前简化版主线并不依赖它跑通
