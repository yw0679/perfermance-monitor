# 项目描述

**项目描述：** 本项目是通过对分布式架构上各个服务器系统多个指标的监控，并通过 grpc、protobuf 进行网络通信将数据指标传输给管理者服务器，进行记录统计，分析出最优服务器的性能供用户使用，来降低负载服务器的压力，以及统计各个服务器的状态变化预防服务器未知突变原因引起的异常。

**使用的中间件：** c++、grpc、protobuf、mysql、ebpf、cmake

## 项目的主要内容

1. **CPU负载统计：** 使用读取/proc的方式对 cpu 负载情况进行每几分钟的统计。
2. **CPU使用情况统计：** 通过编写内核模块，加载内核模块的方式，获取系统的 cpu 空闲时间、io 等待时间等指标，同时计算出各个指标的**使用率、变化率**
4. **内存信息统计：** 通过读取系统的配置文件获取系统总内存，空闲内存，可用内存和缓存，内存使用率，获取系统整体的内存使用情况。
5. **网络信息统计：** 使用 ebpf 中间件，获取内核协议栈中网卡发送/接受数据包的字节数，计算出发送/接受的速率，以及速率的变化。
6. **网络通信对各指标的收集：** 通过 grpc、protobuf 实现网络通信，管理者服务器通过与各个工作者服务器通信进行性能指标收集记录。
7. **性能指标的记录：** 使用关系型数据库 MySQL 对各个指标进行记录，以计算指标的变化率，以防指标有突变行为，来快速反应服务器问题，快速排查。
8. **最优服务器的选择：** 通过不同使用场景，对各个指标赋予权值，来计算出目前性能最优的工作者服务器，供用户使用。
9. **性能验证：** 通过使用 stress、iperf 等模拟不同的系统情况，查看各个指标的数值，来分析在不同指标数值下系统的性能情况，以便日后分析。

## 个人收获

1. 熟悉内核模块的编写，能够为内核编写特定的功能
2. 熟悉使用 grpc 等网络通信框架
3. 对 ebpf 熟悉学习，可以与内核进行数据的交互
