**项目描述**：本项目是一个 manager / worker 分布式性能监控系统，通过对各个服务器系统多个指标的监控，并通过 grpc、protobuf 进行网络通信将数据指标传输给管理者服务器，进行记录统计、查询分析，选出当前更优服务器供用户使用，来降低负载服务器的压力，以及统计各个服务器的状态变化预防服务器未知突变原因引起的异常。

**使用的中间件**:c++、grpc、protobuf、mysql、ebpf、libbpf、cmake

**项目的主要内容**:

1.**系统资源统计**：通过读取 /proc、/sys 等系统虚拟文件完成 CPU、内存、磁盘 IO 等核心指标采集，并计算使用率、变化率等统计结果，为后续性能分析、最优服务器选择和异常排查提供数据支持。

2.**网络信息统计**：使用 ebpf、libbpf，在 TC ingress / egress 挂点获取内核协议栈中网卡发送 / 接受数据包的字节数和包数，计算出发送 / 接受速率，以及收发包速率。

3.**网络通信对各指标的收集**：通过 grpc、protobuf 实现网络通信，由工作者服务器主动向管理者服务器推送主机标识和性能指标，管理者服务器统一进行接收处理和记录。

4.**性能指标的记录**：使用关系型数据库 MySQL 对各个指标进行记录，以便快速反应服务器问题并进行排查。

5.**最优服务器的选择**：通过不同使用场景，对 CPU、内存、负载、磁盘、网络等指标赋予权值，计算出服务器综合评分，并结合平滑加权轮询选择目前性能更优的工作者服务器，供用户使用。

6.**性能压测与稳定性验证**：通过使用 stress、iperf 等模拟不同的系统情况，查看各个指标的的数值，来分析在不同指标数值下系统的性能情况，以便日后分析。

7.**项目的部署上线**：通过 docker, 编写部署脚本，使用镜像模式键在多台服务器上进行环境的部署。

**个人收获：**

1.熟悉使用 grpc、protobuf 进行 C++ 服务间网络通信和查询接口设计。

2.对 ebpf、libbpf 在 TC 挂点进行网络流量统计，以及 mysql、cmake 的工程整合有了基本理解。
