**项目描述**：本项目旨在对分布式集群的节点运行状态与网络链路进行全方位实时监控。通过 C++ 独立实现 Agent 端多维度核心指标采集，利用 gRPC 将数据高频上报至管理端。系统不仅创新性地引入了 LLM 接口进行智能诊断，更针对高并发场景设计了防倾斜的路由调度算法，并通过严格的微基准压测与故障注入验证了系统的高可用性。

**使用的技术栈：**C++11/14、gRPC、Protobuf、MySQL、Raw Socket、大模型 API

****项目的主要内容：****

1. ****多维度系统指标采集与计算：****** 通过解析 Linux /proc 伪文件系统，在内存中维护时间滑窗。通过差值公式精确计算 CPU 真实使用率； 计算网络吞吐量；实时获取物理内存与 Swap 分配状态。**
2. ****轻量级网络链路诊断（RTT）：****** 基于 C++ 原始套接字 (Raw Socket) 独立封装 ICMP 协议，实现轻量级异步 Ping 工具。手动计算 Checksum 并解析 ICMP 回显应答，定时探测 Worker 节点与外部依赖服务的网络连通性及 RTT 延迟。**
3. ****高性能 RPC 数据上报：****** 统一定义 Protobuf 数据结构，使用 gRPC 框架实现 Agent 端到 Server 端的跨节点长连接通信，保障监控指标与网络 RTT 状态的高频并发上报。**
4. ****LLM 智能辅助诊断分析：****** 服务端对全量指标落库 MySQL 并设立阈值报警机制。当检测到 CPU 突增或网络 RTT 异常飙升时，自动提取上下文指标封装为标准化 Prompt，通过 API 异步调用大模型进行根因初步分析，并将排障建议输出至告警日志。**
5. ****高并发场景下的防倾斜路由调度：****** 为避免单纯返回“绝对最优节点”导致的流量倾斜与“羊群效应”，设计并实现了******平滑加权轮询调度******算法，综合 CPU、内存与 RTT 延迟动态生成权重输出最优节点。**
6. ****极限场景压测与并发调优：****** 针对 Server 端“读多写少”的状态访问瓶颈，编写 C++ ******多线程微基准测试******,并使用******ghz******进行测试。 同时利用 **`stress`** 与 **`iperf`** 注入高负载，验证了 AI 报警链路与容灾调度的准确性。**
